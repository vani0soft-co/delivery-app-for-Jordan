# Contributing to Delivery App for Jordan

## English

### Welcome to Our Community! ๐

Thank you for your interest in contributing to the Delivery App for Jordan project. We welcome contributions from everyone and are excited to work with you!

### How to Contribute

#### 1. Reporting Issues
- Check if the issue already exists in the [Issues](../../issues) section
- Provide a clear and descriptive title
- Include a detailed description of the problem
- Add steps to reproduce the issue
- Attach screenshots or error logs if applicable

#### 2. Suggesting Enhancements
- Use a clear and descriptive title
- Provide a detailed description of the suggested enhancement
- Explain why this enhancement would be useful
- List any related issues or pull requests

#### 3. Submitting Pull Requests

**Before you start:**
- Fork the repository
- Create a new branch from `main` with a descriptive name
- Follow the code style guidelines below

**Making your changes:**
1. Make your code changes
2. Ensure all tests pass
3. Add tests for new features or bug fixes
4. Update documentation as needed
5. Commit with clear, concise messages

**Submitting your PR:**
- Push your branch to your fork
- Open a pull request against the `main` branch
- Fill out the pull request template completely
- Link related issues using `#issue_number`
- Request a review from maintainers

### Code Style Guidelines

- Use consistent indentation (2 or 4 spaces, as per project conventions)
- Write meaningful variable and function names
- Add comments for complex logic
- Follow the existing code structure and patterns
- Remove unused imports and variables
- Write descriptive commit messages

### Testing

- Write unit tests for new features
- Ensure all existing tests pass before submitting
- Maintain or improve code coverage
- Test edge cases and error scenarios

### Documentation

- Update README.md if your changes affect project setup or usage
- Add inline comments for complex algorithms
- Update CHANGELOG if making significant changes
- Include examples for new features

### Code of Conduct

We are committed to providing a welcoming and inclusive environment. Please:
- Be respectful and professional
- Include people of all backgrounds and experience levels
- Assume good intentions
- Address conflicts constructively
- Report violations to the maintainers

### Getting Help

- Check existing [documentation](../../wiki)
- Review closed issues and pull requests for similar questions
- Ask questions in pull request discussions
- Contact maintainers if you need guidance
